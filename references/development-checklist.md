# 开发完成自检清单

## 生成范围

- [ ] 没有误生成项目已有 Layout。
- [ ] 没有误生成 Sidebar / Header / Logo / Menu。
- [ ] content 区域结构与 Figma 业务模块一致。

## 字体与变量

- [ ] 没有在组件内写 `font-family`。
- [ ] 字重映射为 400 / 500 / 600。
- [ ] 颜色类样式以 Figma 实际色值为最高依据。
- [ ] 已检查项目已有变量、design token、`custom-figma-constant.less`，只有色值完全一致时才复用变量。
- [ ] 没有把语义相近但色值不同的变量强行用于颜色类样式。
- [ ] 少量局部颜色直接写在当前 class 中，没有使用行内 style。
- [ ] 重复出现或页面级核心颜色已沉淀到 `custom-figma-constant.less`。
- [ ] 新增颜色变量使用 `@figma-` 前缀，没有创建 `@color-*` 这类通用变量。
- [ ] 没有使用 Figma 节点 ID、图层名、组件名直接命名变量。

## 组件与事件

- [ ] 页面负责 router / api / store。
- [ ] 展示组件不直接调接口。
- [ ] 展示组件不直接 `router.push`。
- [ ] emit 事件语义化。
- [ ] 列表、卡片、统计、状态都数据驱动。

## 样式与布局

- [ ] 已识别并解析 `reference.tsx` 中框架化的 `className`。
- [ ] `className` 解析结果已转换成目标项目样式方案，没有原样照搬不兼容的 Figma 生成类名。
- [ ] `className` 解析结果已与 `meta.json`、Figma MCP 原始数据或截图真实渲染结果校验。
- [ ] 组件之间间距由父级控制。
- [ ] 组件内部没有无脑使用 `gap`。
- [ ] 固定区使用 `flex: 0 0 auto`。
- [ ] 弹性区使用 `flex: 1 1 auto` 和 `min-width: 0`。
- [ ] 长文本有 ellipsis。
- [ ] 重要业务长文本有 Tooltip 或 `title`。
- [ ] 没有用 `inline-block` 做主布局。
- [ ] 只有规整宫格才使用 grid。

## 响应式

- [ ] 声明了 1440 / 1024 / 768 / 520 / 375 的模块行为。
- [ ] 没有所有模块一刀切纵向排列。
- [ ] 375 是最小设计保障宽度。
- [ ] 520 以下进入手机布局策略。

## 表单与表格

- [ ] 已有全局 Table 时没有手写 table。
- [ ] 表单区分筛选表单和编辑表单。
- [ ] 小屏表单能一行一个。
- [ ] 搜索框 / 日期范围等特殊控件按规则独占一行。

## 视觉验收

- [ ] 输出结构化自检报告。
- [ ] chrome-devtools MCP 可用时已截图检查。
- [ ] 不可用时已说明降级。
