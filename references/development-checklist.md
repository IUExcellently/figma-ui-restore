# 开发完成自检清单

## 生成范围

- [ ] 没有误生成项目已有 Layout。
- [ ] 没有误生成 Sidebar / Header / Logo / Menu。
- [ ] content 区域结构与 Figma 业务模块一致。

## 字体与变量

- [ ] 没有在组件内写 `font-family`。
- [ ] 字重映射为 400 / 500 / 600。
- [ ] 颜色优先使用已有变量。
- [ ] 新增变量放在 `custom-figma-constant.less` 或项目约定文件。
- [ ] 没有 Figma 图层 ID 式变量名。

## 组件与事件

- [ ] 页面负责 router / api / store。
- [ ] 展示组件不直接调接口。
- [ ] 展示组件不直接 `router.push`。
- [ ] emit 事件语义化。
- [ ] 列表、卡片、统计、状态都数据驱动。

## 样式与布局

- [ ] 组件之间间距由父级控制。
- [ ] 组件内部没有无脑使用 `gap`。
- [ ] 固定区使用 `flex: 0 0 auto`。
- [ ] 弹性区使用 `flex: 1 1 auto` 和 `min-width: 0`。
- [ ] 长文本有 ellipsis。
- [ ] 重要业务长文本有 Tooltip 或 `title`。
- [ ] 没有用 `inline-block` 做主布局。
- [ ] 只有规整宫格才使用 grid。

## 响应式

- [ ] 声明了 1440 / 1024 / 768 / 520 / 375 的模块行为。
- [ ] 没有所有模块一刀切纵向排列。
- [ ] 375 是最小设计保障宽度。
- [ ] 520 以下进入手机布局策略。

## 表单与表格

- [ ] 已有全局 Table 时没有手写 table。
- [ ] 表单区分筛选表单和编辑表单。
- [ ] 小屏表单能一行一个。
- [ ] 搜索框 / 日期范围等特殊控件按规则独占一行。

## 视觉验收

- [ ] 输出结构化自检报告。
- [ ] chrome-devtools MCP 可用时已截图检查。
- [ ] 不可用时已说明降级。
