# 05 · 响应式与断点规则

## 全局断点

```less
@screen-xxl: 1440px; // Figma 基准稿 / 大屏桌面
@screen-lg: 1024px;  // 窄桌面 / 横向平板
@screen-md: 768px;   // iPad / 平板竖屏临界
@screen-sm: 520px;   // 手机布局策略临界
@screen-xs: 375px;   // 最小设计保障宽度
```

## 断点含义

- 1440：设计基准，不一定每次都写媒体查询。
- 1024：常规桌面 / 窄桌面。
- 768：平板 / 后台窄内容区。
- 520：进入手机布局策略。
- 375：项目最小设计保障宽度。

375 以下不作为主要适配目标，允许降级或横向滚动。

## 核心原则

响应式不是全局一刀切。

每个业务模块必须单独声明：

- 1440 怎么排
- 1024 怎么排
- 768 怎么排
- 520 怎么排
- 375 怎么排
- 是否允许换行
- 哪些元素固定
- 哪些元素弹性
- 哪些文本省略

## 后台页面优先看 content 宽度

后台页面有 Sidebar / Header，viewport 不等于 content 可用宽度。

如果项目支持 container query，优先基于内容容器判断。

如果不支持，再使用项目统一媒体查询变量。

## 示例策略

### 欢迎模块

- 始终左右布局。
- 左侧上下布局。
- 右侧不收缩。
- 不换行。
- 左侧长文本 ellipsis。

### 快捷入口卡片

- 大屏横向排列。
- 宽度不足时换行或纵向排列。
- 每个卡片可 `flex: 1 1 360px`。

### 预警 + 统计卡

- 大屏：预警主区域 + 右侧统计卡一行。
- 中屏：统计卡整体换到第二行，两个卡均分。
- 小屏：全部纵向。
