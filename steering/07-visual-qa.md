# 07 · 视觉自检与 chrome-devtools MCP 验收

## 验收分层

1. 结构化自检：不依赖浏览器。
2. 浏览器截图验收：如果 chrome-devtools MCP 可用。
3. 人工最终确认。

## 结构化自检内容

开发完成后必须输出自检报告，检查：

- 是否误生成 Layout / Sidebar / Header。
- 是否忽略 Figma font-family。
- 是否使用变量而非裸色值。
- 是否复用项目已有组件。
- 组件之间间距是否由父级控制。
- 组件内部间距是否按设计意图处理。
- 是否有响应式策略。
- 是否覆盖 1440 / 1024 / 768 / 520 / 375。
- 长文本是否 ellipsis + Tooltip。
- flex 子项是否有 `min-width: 0`。
- 组件是否只 emit 语义事件。
- 是否避免组件内部直接 router / api。

## chrome-devtools MCP 可用时

必须执行浏览器截图验收：

1. 打开本地页面地址。
2. 设置 viewport：1440、1024、768、520、375。
3. 每个尺寸截图一次。
4. 检查横向滚动、文字溢出、模块重叠、换行错误、间距异常、组件变形。
5. 发现问题后修复。
6. 修复后重新截图验证。

## 不可用时

如果 chrome-devtools MCP 不可用，必须明确说明：

```text
当前环境无法调用 chrome-devtools MCP，已降级为结构化自检，未完成真实浏览器截图验收。
```

不得声称已经完成截图检查。

## 不做强制像素级 diff

第一版不强制做像素级 diff。

1440 用于对照 Figma 基准稿；其他尺寸主要检查响应式策略是否合理。
