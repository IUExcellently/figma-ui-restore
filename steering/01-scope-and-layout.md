# 01 · 页面类型与 Layout 边界

## 第一版范围

只处理后台管理系统页面。前台展示页、详情页、大屏看板暂不处理。

## 页面类型判断

拿到 Figma 页面后，先判断它属于：

1. 后台管理页面
2. 前台展示页面
3. 移动端页面
4. 大屏 / 看板页面
5. 嵌入式业务模块

若不是后台管理页面，停止当前工作流，提示用户第一版暂不覆盖。

## 后台 Layout 形态判断

后台页面常见结构：

1. 左侧菜单 + 右侧内容
2. 顶部 Header + 下方内容
3. 顶部 Header + 左侧菜单 + 右侧内容
4. 其他特殊结构

## 必问问题

在写代码前必须确认：

```text
当前项目是否已经存在后台 Layout / Sidebar / Header / Menu？
```

### 如果已有 Layout

- 不生成 Sidebar。
- 不生成 Header。
- 不生成 Logo。
- 不生成 Menu。
- 只还原 content 区域。
- Figma 里的全局壳只作为视觉参考。

### 如果没有 Layout

询问用户：

```text
是否需要基于当前 Figma 页面生成通用后台 Layout？
```

用户确认前，不得擅自生成后台壳。

## 核心原则

后台系统不追求每个页面独立成完整视觉稿，而追求全局一致性。

AI 应优先尊重项目已有 Layout，而不是优先尊重单张 Figma 图里的完整画面。
