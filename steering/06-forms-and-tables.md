# 06 · 表单与表格边界

## 表格类页面

如果项目已有全局 Table 组件：

- 不根据 Figma 手写 table。
- 不复制 Figma 表格内部 DOM。
- 只识别表格所在容器、标题区、操作区、搜索区、分页区。
- 表格列配置、接口字段、操作列由页面逻辑处理。
- 使用项目已有 Table 组件完成渲染。

## 表单类型判断

先判断表单属于：

1. 筛选表单
2. 编辑 / 新增表单
3. 弹窗表单
4. 详情条件表单

不要用同一套规则套所有 form。

## 编辑 / 新增表单

PC 默认一行两个表单项。

小于 768 或 520 后默认一行一个并换行。

## 筛选表单

筛选表单追求紧凑，常见行为：

- 普通输入框可多列排列。
- 搜索输入框可独占一行。
- 日期范围选择器可独占一行。
- 操作按钮可跟随末尾，空间不足时换行。
- 支持展开 / 收起时，不要破坏行高和对齐。

## 特殊控件独占规则

以下控件可根据宽度独占一行：

- 搜索输入框
- 日期范围选择器
- 长文本输入
- 地址 / 组织 / 人员选择器
- 联级选择器
- 多选标签较多的 Select

## Label 与校验

- PC 上保持 label 对齐。
- 小屏下一行一个，label 和 input 的布局按项目既有规范。
- 校验错误信息不能遮挡或挤压相邻项。
