# mini-program · 文本、省略与换行规则

## 单行省略

单行省略必须同时检查：

```css
.text {
  overflow: hidden;
  text-overflow: ellipsis;
  white-space: nowrap;
}
```

如果文本处于 flex 子项中，必须检查是否需要：

```css
.text {
  min-width: 0;
}
```

## 多行省略

多行省略必须确认目标基础库、目标运行时和项目兼容方案。

不得默认所有平台都完全支持同一种多行省略写法。

## 业务文本不可盲目省略

以下文本不能只做视觉省略：

- 用户名
- 地址
- 订单号
- 审批意见
- 异常原因
- 错误提示
- 评价内容
- 业务说明

需要根据业务提供完整展示方式，例如：

- 点击展开
- 详情页查看
- 弹窗展示
- 复制完整内容
- title / tooltip 的等价方案

## 不机械照抄固定高度

Figma 中固定高度文本容器，不代表代码里也必须固定高度。

生成代码时需要判断：

- 真实数据是否可能多行。
- 文本是否来自接口。
- 文案是否支持国际化。
- 字号是否会随系统设置变化。
- 是否会影响卡片高度。
