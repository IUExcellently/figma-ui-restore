# mini-program · 性能、包体与长列表规则

## 包体与资源预算

新增本地图片、背景图、装饰图前必须评估包体影响。

规则：

- 不得默认把页面专属大图放入主包。
- tabBar 页面、首页首屏资源才考虑放入主包。
- 非首屏页面资源优先放入对应分包或 CDN。
- 大背景图、插画、运营图必须确认是否走 CDN。
- 不写死总包大小限制；以当前平台官方文档、项目配置和上传工具校验结果为准。
- 开发后自检报告必须说明新增资源数量、资源位置、是否可能影响主包体积。

## 本地背景图

小程序本地图片不应默认通过 WXSS `background-image: url(...)` 引用。

推荐策略：

- 本地背景图优先使用 `image` 组件绝对定位模拟背景。
- 网络图片可按项目规范使用 `background-image`。
- 小图标优先使用项目现有 iconfont / svg / image 方案。
- 大装饰图必须评估压缩、复用和包体影响。

## 图片加载体验

需要考虑：

- 图片未加载时的占位。
- 图片加载失败时的兜底。
- 首屏图片闪烁。
- 网络图加载慢导致布局跳动。
- 多页面复用资源是否重复引入。

## setData / 状态更新

不得在以下场景频繁更新大对象：

- 页面滚动
- 输入框输入
- 动画执行中
- 倒计时
- 长列表滚动
- 拖拽排序
- 手势移动

规则：

- 只更新必要字段。
- 不整体替换超大数组。
- 不把接口原始大对象完整塞入视图层状态。
- 展示组件优先接收精简后的 ViewModel。
- 复杂数据转换放在请求完成后或 computed / memo 层处理，不在模板里反复计算。

## 长列表

长列表必须确认：

- 是否分页。
- 是否触底加载。
- 是否下拉刷新。
- 是否有 loading 状态。
- 是否有 empty 状态。
- 是否有 error / retry 状态。
- 是否需要虚拟列表。
- item 是否有稳定 key。

不得一次性渲染超大 mock 数据。

## 页面生命周期

必须按页面生命周期处理请求和清理：

- 页面进入时是否请求。
- 页面显示时是否刷新。
- 页面隐藏时是否暂停计时器 / 监听。
- 页面卸载时是否清理定时器、事件监听、异步任务。
