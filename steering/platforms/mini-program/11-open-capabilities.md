# mini-program · 开放能力、授权与安全规则

## 开放能力不能当普通按钮处理

如果 UI 涉及以下能力，不得只按普通按钮生成：

- 获取手机号
- 获取头像昵称
- 分享
- 订阅消息
- 位置权限
- 相机
- 相册
- 蓝牙
- NFC
- 文件
- 支付
- 客服
- web-view
- 地图导航

实现前必须确认：

- 项目是否已有能力封装。
- 是否需要用户授权弹窗。
- 是否需要隐私协议配置。
- 是否需要 `privacy.json`。
- 拒绝授权时如何展示。
- 授权失败时如何重试。
- 是否需要服务端接口配合。

## 分享

涉及分享时需要确认：

- 是否是页面分享。
- 是否是按钮分享。
- 分享标题。
- 分享路径。
- 分享图片。
- 是否带业务参数。
- 参数是否需要加密或短链处理。

## 订阅消息

涉及订阅消息时需要确认：

- 模板 ID 来源。
- 触发时机。
- 用户拒绝后的兜底。
- 是否需要服务端发送。
- 是否已有项目封装。

## 支付

涉及支付时不得生成假支付逻辑。

必须确认：

- 后端下单接口。
- 支付参数来源。
- 成功回调。
- 失败回调。
- 取消支付处理。
- 订单状态轮询或查询策略。

## 安全

不得在小程序前端生成或保存：

- appSecret
- 长期 token
- 私钥
- 后台管理密钥
- 服务端接口签名密钥

敏感逻辑必须放到服务端。
